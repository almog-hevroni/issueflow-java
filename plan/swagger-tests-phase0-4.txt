2) Login Prerequisite (Important Blocker)
========================================================================

Current code does NOT create a seeded login user automatically.
POST /auth/login is public, but all other APIs are protected.
Therefore, if you have no existing user row, login cannot succeed.
User creation now supports optional password:
- if password is provided in POST /users, that password is used.
- if password is omitted, the configured fallback initial password is used:
  issueflow.security.initial-password (env: ISSUEFLOW_SECURITY_INITIAL_PASSWORD,
  default: Password123!).

2.1 Simplest local solution (insert one user directly in DB)
Run SQL in PostgreSQL:

  CREATE EXTENSION IF NOT EXISTS pgcrypto;

  INSERT INTO users (username, email, full_name, role, password_hash)
  VALUES (
    'swagger_admin_local',
    'swagger_admin_local@example.com',
    'Swagger Local Admin',
    'ADMIN',
    crypt('Password123!', gen_salt('bf'))
  )
  ON CONFLICT (username) DO NOTHING;

2.2 Verify bootstrap user
  SELECT id, username, role FROM users WHERE username = 'swagger_admin_local';

Save:
  <ADMIN_ID>

- Public:
  POST /auth/login
  /v3/api-docs/**
  /swagger-ui/**
  /swagger-ui.html
  /actuator/health

- Additional role restrictions:
  - GET /tickets/deleted -> ADMIN only
  - POST /tickets/{id}/restore -> ADMIN only

13) DB Verification Checklist
========================================================================

Users:
  SELECT id, username, email, role FROM users ORDER BY id DESC;

Projects:
  SELECT id, name, owner_id, deleted_at FROM projects ORDER BY id DESC;

Tickets:
  SELECT id, project_id, assignee_id, status, priority, due_date, is_overdue, deleted_at
  FROM tickets ORDER BY id DESC;

Dependencies:
  SELECT ticket_id, blocked_by_ticket_id, created_at FROM ticket_dependencies ORDER BY created_at DESC;

Comments:
  SELECT id, ticket_id, author_id, content FROM comments ORDER BY id DESC;

Comment mentions:
  SELECT comment_id, mentioned_user_id, created_at FROM comment_mentions ORDER BY created_at DESC;

Attachments:
  SELECT id, ticket_id, uploaded_by_id, file_name, content_type, size_bytes, storage_path
  FROM attachments ORDER BY id DESC;

Audit:
  SELECT id, action, entity_type, entity_id, actor_type, actor_user_id, created_at
  FROM audit_logs ORDER BY id DESC;

Revoked tokens:
  SELECT id, jti, revoked_at, expires_at FROM revoked_tokens ORDER BY id DESC;
